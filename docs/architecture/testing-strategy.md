# Testing Strategy

## Testing Pyramid

- **Unit Tests (60%):** Components, services, utilities
- **Integration Tests (30%):** API endpoints, database queries
- **E2E Tests (10%):** Critical user journeys

## Test Coverage Requirements

| Component Type | Coverage Target |
| -------------- | --------------- |
| Services       | 80%+            |
| Repositories   | 90%+            |
| API Routes     | 80%+            |
| UI Components  | 70%+            |
