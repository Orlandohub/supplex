## Story

**As a** developer,  
**I want** an email notification system for workflow events,  
**so that** users are notified of actions requiring their attention.

---

## Acceptance Criteria

1. Email service integrated (Resend.com) with API key configured
2. Email templates created for workflow events: Workflow submitted, Approval needed, Workflow approved, Workflow rejected, Stage advanced
3. Email template uses Supplex branding with logo, consistent styling
4. Notification preferences configured at tenant level: Enable/disable email notifications per event type
5. User preferences page allows individual users to opt-in/out of specific notification types
6. Emails include: Actionable subject line, Recipient name, Summary of action, Direct deep link to workflow, Sender/actor name
7. Background job worker (BullMQ) processes email sending asynchronously
8. Failed emails are retried up to 3 times with exponential backoff
9. Email send status tracked in database: Pending, Sent, Failed, Bounced
10. Email logs accessible to admins for debugging (last 30 days)
11. Rate limiting prevents email spam (max 10 emails per user per hour)
12. Unsubscribe link included in all emails (compliance with CAN-SPAM)
13. Email deliverability monitored (bounce rate, open rate) in admin dashboard (Phase 2: just log for now)

### Coding Standards Compliance

[Source: architecture/coding-standards.md#Critical-Fullstack-Rules]

**CRITICAL RULES:**

1. **ElysiaJS Route Organization:**
   - Only parent aggregator (`index.ts`) has prefix
   - Child routes must NOT have prefixes
   - Use consistent parameter names (`:token` for unsubscribe)

2. **ElysiaJS Validation:**
   - Use TypeBox (`t.*`) exclusively, never Zod (`z.*`)
   - For enums, use `t.Union([t.Literal(...), ...])`

3. **Authentication Middleware:**
   - Use `authenticate` middleware directly on routes
   - Perform role checks inside handlers with null checks: `if (!user?.role || ...)`

4. **Database Schema:**
   - Verify field names against actual schema files
   - Add null checks for joined data: `joinedTable?.field`

5. **Remix Data Loading:**
   - ALL data fetching in loaders (server-side), never useEffect
   - Use `Promise.all` for parallel data fetching
   - Pass data via props, not by fetching in components

6. **Tenant Isolation:**
   - All database queries must include tenant filter
   - Enforce in all route handlers and services

7. **Type Sharing:**
   - Define types in `packages/types` and import from there

8. **Error Handling:**
   - All API routes must use standard error handler
   - Return proper HTTP status codes (404, 403, 500)

**Email Template Rendering:**
```typescript
import * as path from 'path';
import * as fs from 'fs';

// Template cache to avoid re-reading files on every email
const templateCache = new Map<string, string>();

function loadTemplate(templatePath: string): string {
  if (templateCache.has(templatePath)) {
    return templateCache.get(templatePath)!;
  }
  
  const absolutePath = path.join(__dirname, '../../templates/emails', templatePath);
  const content = fs.readFileSync(absolutePath, 'utf-8');
  templateCache.set(templatePath, content);
  return content;
}

export function renderEmailTemplate(templateName: string, data: Record<string, any>): string {
  // Load templates (cached after first load)
  const baseTemplate = loadTemplate('base.html');
  const contentTemplate = loadTemplate(`${templateName}.html`);

  // Replace placeholders in content template
  let renderedContent = contentTemplate;
  for (const [key, value] of Object.entries(data)) {
    // Escape HTML to prevent injection, convert to string
    const safeValue = String(value).replace(/[<>&"']/g, (c) => {
      const escapeMap: Record<string, string> = {
        '<': '&lt;',
        '>': '&gt;',
        '&': '&amp;',
        '"': '&quot;',
        "'": '&#39;',
      };
      return escapeMap[c];
    });
    renderedContent = renderedContent.replace(new RegExp(`{{${key}}}`, 'g'), safeValue);
  }

  // Insert content into base template
  const finalHtml = baseTemplate.replace('{{CONTENT}}', renderedContent);

  return finalHtml;
}

// Optional: Clear cache in development mode for hot-reloading
export function clearTemplateCache() {
  templateCache.clear();
}
```

### Rate Limiting Logic

**Redis Key Structure:**
```typescript
// Key: email_rate_limit:{user_id}:{hour}
// Value: count (integer)
// TTL: 3600 seconds (1 hour)

async function checkRateLimit(userId: string): Promise<boolean> {
  const hour = new Date().toISOString().slice(0, 13); // 2025-10-25T14
  const key = `email_rate_limit:${userId}:${hour}`;

  const count = await redis.incr(key);
  if (count === 1) {
    await redis.expire(key, 3600); // Set TTL on first increment
  }

  if (count > 10) {
    console.log(`[RATE LIMIT] User ${userId} exceeded 10 emails/hour`);
    return false; // Rate limit exceeded
  }

  return true; // Allow email
}
```

### Notification Preference Check Logic

**Precedence:**
1. Check tenant-level settings first (tenant.settings.emailNotifications)
2. If tenant setting is disabled, skip email (don't check user preferences)
3. If tenant setting is enabled, check user-level preference
4. If user preference is disabled or unsubscribed, skip email

```typescript
async function shouldSendEmail(userId: string, tenantId: string, eventType: EmailEventType): Promise<boolean> {
  // Check tenant setting
  const tenant = await db.query.tenants.findFirst({
    where: eq(tenants.id, tenantId),
  });

  const tenantEnabled = tenant?.settings?.emailNotifications?.[eventType] ?? true;
  if (!tenantEnabled) {
    return false; // Tenant disabled this email type
  }

  // Check user preference
  const userPref = await db.query.userNotificationPreferences.findFirst({
    where: and(
      eq(userNotificationPreferences.userId, userId),
      eq(userNotificationPreferences.eventType, eventType)
    ),
  });

  const userEnabled = userPref?.emailEnabled ?? true; // Default to enabled
  return userEnabled;
}
```
