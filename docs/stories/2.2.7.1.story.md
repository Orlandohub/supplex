## Dev Notes

### Previous Story Context

[Source: Story 2.2.6 - Workflow Template Data Model]

Story 2.2.6 implemented the workflow template tables including a `process_type` field intended to categorize workflows into system-defined types (e.g., 'supplier_qualification', 'sourcing', 'product_lifecycle'). 

User feedback from manual testing revealed this constraint is unnecessary and limits flexibility. Organizations want to create workflows freely using descriptive names without being constrained to predefined types.

### Database Schema Current State

```sql
-- Current workflow_template table (from migration 0012)
CREATE TABLE workflow_template (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    process_type VARCHAR(100) NOT NULL, -- THIS NEEDS TO BE REMOVED
    name VARCHAR(255) NOT NULL,
    description TEXT,
    status form_template_status NOT NULL DEFAULT 'draft',
    created_by UUID NOT NULL REFERENCES users(id) ON DELETE RESTRICT,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMP WITH TIME ZONE
);

-- Index that needs to be dropped
CREATE INDEX idx_workflow_template_tenant_process_status 
    ON workflow_template(tenant_id, process_type, status) 
    WHERE deleted_at IS NULL;

-- Index that should remain
CREATE INDEX idx_workflow_template_tenant_status 
    ON workflow_template(tenant_id, status) 
    WHERE deleted_at IS NULL;
```

### Migration Strategy

Since `process_type` was just added in migration 0012 and Story 2.2.7 is not yet complete, we can safely drop the column. The `idx_workflow_template_tenant_status` index is sufficient for filtering published workflows by tenant.

**Migration SQL:**

```sql
-- Migration: Remove process_type from workflow_template
-- Story: 2.2.7.1 - Remove Process Type Constraint from Workflow Templates
-- Generated: 2026-01-25

-- Drop index that references process_type
DROP INDEX IF EXISTS idx_workflow_template_tenant_process_status;

-- Drop process_type column
ALTER TABLE workflow_template DROP COLUMN IF EXISTS process_type;

-- Rollback Instructions (if needed):

-- UPDATE workflow_template SET process_type = 'general' WHERE process_type IS NULL;
-- ALTER TABLE workflow_template ALTER COLUMN process_type SET NOT NULL;
-- CREATE INDEX idx_workflow_template_tenant_process_status 
--     ON workflow_template(tenant_id, process_type, status) 
--     WHERE deleted_at IS NULL;
```
