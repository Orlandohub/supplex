# Story 2.1.8: Task Assignment on Qualification Process Start

<!-- Powered by BMAD™ Core -->

## Status

**Draft**

Date Created: December 22, 2025

## Story

**As the** system,  
**I want** the qualification task assigned to the correct user when a qualification process starts,  
**so that** the right person is responsible for completing the qualification workflow.

---

## Acceptance Criteria

1. If the supplier has a contact user, the task is assigned to the `supplier_user`
2. If the supplier has no contact user, the task is assigned to the **procurement_manager** of the tenant to whom this supplier belongs
3. Exactly one task is created per qualification process start
4. The task listed on the task link page links to the qualification information page

**Qualification Stages Model:**
[Source: packages/db/src/schema/qualification-stages.ts]

```typescript
export const qualificationStages = pgTable("qualification_stages", {
  id: uuid("id").primaryKey().defaultRandom(),
  workflowId: uuid("workflow_id").notNull().references(() => qualificationProcess.id, { onDelete: "cascade" }),
  stageNumber: integer("stage_number").notNull(), // 0 = initial task, 1-3 = review stages
  stageName: varchar("stage_name", { length: 100 }).notNull(),
  assignedTo: uuid("assigned_to").notNull().references(() => users.id, { onDelete: "restrict" }),
  status: varchar("status", { length: 50 }).notNull().default("Pending"), // Pending, Approved, Rejected, Completed
  reviewedBy: uuid("reviewed_by").references(() => users.id, { onDelete: "restrict" }),
  reviewedDate: timestamp("reviewed_date", { withTimezone: true, mode: "date" }),
  comments: text("comments"),
  attachments: jsonb("attachments").notNull().default([]),
  createdAt: timestamp("created_at", { withTimezone: true, mode: "date" }).notNull().defaultNow(),
  updatedAt: timestamp("updated_at", { withTimezone: true, mode: "date" }).notNull().defaultNow(),
  deletedAt: timestamp("deleted_at", { withTimezone: true, mode: "date" }),
});
```

**Key Points:**
- `stageNumber = 0` will represent the initial document upload task
- `stageNumber = 1, 2, 3` represent review stages (existing implementation)
- `assignedTo` field supports any user role including `supplier_user`
- Status values: "Pending", "Approved", "Rejected", "Completed"
