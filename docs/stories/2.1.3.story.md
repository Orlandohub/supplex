## Story

**As a** user,  
**I want** to be prevented from signing in when my account is `deactivated`,  
**so that** I receive a clear explanation and know how to contact my company's admin.

---

## Acceptance Criteria

1. Users marked as `deactivated` (is_active = false) cannot authenticate
2. Authentication fails before session or token creation
3. The following message is displayed on login attempt:

```
Your user has been deactivated, please contact your company's admin:
[admin_name]
[admin_email]
```

4. Active users are not affected

### Technical Implementation Details

#### Backend Implementation (Defense-in-Depth Middleware Check)

[Source: architecture/coding-standards.md]

**Purpose:** Catch deactivated users who were deactivated AFTER logging in (while they have an active session).

**Implementation Location:** Add check in `apps/api/src/lib/rbac/middleware.ts` after JWT validation (Task 2)

**Drizzle ORM Query Pattern:**
```typescript
import { db } from "../db";
import { users } from "@supplex/db/schema";
import { eq, and } from "drizzle-orm";

// Query user record with tenant isolation
const [userRecord] = await db
  .select({
    isActive: users.isActive,
    fullName: users.fullName,
    tenantId: users.tenantId,
  })
  .from(users)
  .where(and(
    eq(users.id, user.id),
    eq(users.tenantId, tenantId)
  ))
  .limit(1);

// Check if user is deactivated
if (!userRecord?.isActive) {
  console.log("[AUTH MIDDLEWARE] User is deactivated");
  
  // Query for active admin in same tenant
  const [adminUser] = await db
    .select({ fullName: users.fullName, email: users.email })
    .from(users)
    .where(and(
      eq(users.tenantId, tenantId),
      eq(users.role, "admin"),
      eq(users.isActive, true)
    ))
    .limit(1);
  
  const adminInfo = adminUser 
    ? `${adminUser.fullName}\n${adminUser.email}`
    : "your company's admin";
  
  set.status = 401;
  throw new Error(
    JSON.stringify({
      error: {
        code: "USER_DEACTIVATED",
        message: `Your user has been deactivated, please contact your company's admin:\n${adminInfo}`,
        timestamp: new Date().toISOString(),
      },
    })
  );
}
```

**Placement in middleware.ts:**
- After JWT validation with Supabase (line ~77)
- After extracting role and tenantId (line ~122-126)
- Before returning user context to route handlers (line ~140+)

#### Frontend Implementation

[Source: architecture/coding-standards.md - Remix Patterns]

**Check is_active After Supabase Login in useAuth.ts:**

**Current Authentication Flow:**
1. User submits credentials → `useAuth.signIn()` → `supabase.auth.signInWithPassword()` (line 77)
2. Supabase validates credentials and creates session (JWT token issued)
3. Frontend fetches user record from database (line 89-93) ✅ **ALREADY IMPLEMENTED**
4. Frontend sets auth state (line 99)

**Implementation: Add is_active Check (Task 1)**

The frontend already fetches the user record after Supabase authentication (line 89-93). We need to add the `is_active` check between fetching the user record and setting auth state:

```typescript
// After successful Supabase authentication (line 87)
if (data.user && data.session) {
  // Fetch user record from database (EXISTING CODE - line 89-93)
  const { data: userRecord, error: userError } = await supabase
    .from("users")
    .select("*")
    .eq("id", data.user.id)
    .single();

  if (userError) {
    console.error("Error fetching user record:", userError);
    // EXISTING ERROR HANDLING (line 95-97)
  }

  // ===== NEW CODE: Check if user is deactivated =====
  if (userRecord && !userRecord.is_active) {
    // Fetch admin contact info
    const { data: adminUsers } = await supabase
      .from("users")
      .select("full_name, email")
      .eq("tenant_id", userRecord.tenant_id)
      .eq("role", "admin")
      .eq("is_active", true)
      .limit(1);

    const adminUser = adminUsers?.[0];
    const adminInfo = adminUser 
      ? `${adminUser.full_name}\n${adminUser.email}`
      : "your company's admin";

    // Sign out immediately (revoke the session that was just created)
    await supabase.auth.signOut();

    set({ isLoading: false });
    return {
      success: false,
      error: `Your user has been deactivated, please contact your company's admin:\n${adminInfo}`,
    };
  }
  // ===== END NEW CODE =====

  // EXISTING CODE: User is active, proceed with normal flow (line 99)
  get().setAuth(data.user, data.session, userRecord);
  // ... rest of existing code
}
```

**Key Points:**
- Check happens AFTER Supabase auth but BEFORE storing session in app state
- Session is immediately revoked via `signOut()` if user is deactivated
- Error message preserves line breaks (`\n`) for proper display
- No session stored = user never reaches authenticated state

### Technical Constraints

- **Authentication Middleware:** Use `authenticate` middleware directly on routes
