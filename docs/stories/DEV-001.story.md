## Story

**As a** developer,  
**I want** a quick login feature in development mode,  
**so that** I can efficiently test with different user types (tenant users and supplier users) without repeatedly entering credentials.

---

## Context

### Current Testing Challenge

Manual testing in development environments requires repeatedly entering credentials for different user scenarios:
- Testing as different tenants
- Testing as different roles (admin, procurement_manager, quality_manager, supplier_user)
- Testing supplier-specific workflows
- Testing cross-tenant scenarios

**Current Workflow (Inefficient):**
```
1. Navigate to login page
2. Type email: john@acme.com
3. Type password: ********
4. Click Sign In
5. Test feature
6. Log out
7. Repeat for different user...
```

**Time Impact**: ~30 seconds per login × 20+ logins per day = **10 minutes daily overhead**

### Proposed Solution

Add a development-only "Quick Login" UI that provides:
- Dropdown to select tenant
- Dropdown to select user (tenant users + supplier users)
- User roles visible in dropdown labels
- One-click login without password entry

**New Workflow (Efficient):**
```
1. Navigate to login page
2. Select tenant from dropdown
3. Select user from dropdown (shows role)
4. Click "Quick Login"
5. Test feature
```

**Time Impact**: ~5 seconds per login × 20+ logins per day = **~2 minutes daily** (80% time savings)

### Industry Standards

This pattern is common in development tools:
- **Auth0**: Dev login panel in test environments
- **Firebase**: Test users dropdown in emulator
- **Supabase Studio**: Quick user impersonation
- **Clerk**: Dev mode test users

### JWT Generation Options

**Option 1: Supabase Admin (Recommended)**
```typescript
const { data, error } = await supabaseAdmin.auth.admin.generateLink({
  type: 'magiclink',
  email: user.email,
});
// Returns access_token
```

**Option 2: Custom JWT Signing**
```typescript
import { SignJWT } from 'jose';

const jwt = await new SignJWT({
  sub: user.id,
  email: user.email,
  user_metadata: {
    role: user.role,
    tenant_id: user.tenantId,
  },
})
  .setProtectedHeader({ alg: 'HS256' })
  .setIssuedAt()
  .setExpirationTime('1h')
  .sign(new TextEncoder().encode(config.jwt.secret));
```

### User Query Example

```typescript
// Query all users grouped by tenant
const tenants = await db.select().from(tenantsTable);

const result = await Promise.all(
  tenants.map(async (tenant) => {
    // Tenant users
    const users = await db
      .select()
      .from(usersTable)
      .where(eq(usersTable.tenantId, tenant.id))
      .where(eq(usersTable.isActive, true));

    // Supplier users for this tenant's suppliers
    const suppliers = await db
      .select()
      .from(suppliersTable)
      .where(eq(suppliersTable.tenantId, tenant.id));

    const supplierUserIds = suppliers.map((s) => s.primaryContactId);
    const supplierUsers = await db
      .select()
      .from(usersTable)
      .where(inArray(usersTable.id, supplierUserIds));

    return {
      id: tenant.id,
      name: tenant.name,
      users,
      supplierUsers,
    };
  })
);
```
