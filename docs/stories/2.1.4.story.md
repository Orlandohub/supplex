## Story

**As a** tenant_user,  
**I want** suppliers to access the platform through a restricted user role,  
**so that** they can manage only their own information and tasks.

---

## Acceptance Criteria

1. A new role `supplier_user` exists
2. `supplier_user` can:
   * View and edit its own supplier information
   * View its own task list
   * Manage personal notification settings
3. `supplier_user` cannot access other suppliers or tenant configuration

### Technical Constraints

[Source: architecture/coding-standards.md]

**Critical Rules:**
- **Type Sharing:** Always define types in `packages/types` and import from there
- **Tenant Isolation:** All database queries must include tenant filter (supplier_user will be scoped to single supplier in future stories)
- **Authentication Middleware:** Use `authenticate` middleware directly on routes
- **Role Checks:** Perform role checks inside handlers with null checks: `if (!user?.role || ...)`
- **Error Handling:** All API routes must use standard error handler pattern
- **Validation:** Use TypeBox (`t.*`) for ElysiaJS route validation, Zod (`z.*`) for shared types

**Security Considerations:**
- **Principle of Least Privilege:** supplier_user has minimal permissions (only own supplier, own tasks, notification settings)
- **No Cross-Supplier Access:** supplier_user cannot view or edit other suppliers (enforced via permissions)
- **No Tenant Admin Access:** supplier_user cannot access tenant configuration or user management
- **Future Implementation:** API routes for supplier-specific operations will verify user owns the supplier record
- **Task Assignment:** supplier_user can only view tasks assigned to them (enforced in `/my-tasks` query)

**Database Considerations:**
- **No Migration Needed:** `users.role` is VARCHAR(50), already supports "supplier_user" string value
- **No Constraint Validation:** Database does not enforce role enum at DB level (validation happens in application layer)
- **User Metadata:** Role is stored in both `users.role` field AND Supabase Auth `user_metadata` for JWT inclusion
