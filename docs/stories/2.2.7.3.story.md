## Story

**As a** developer,  
**I want** to connect the document template dropdown to the existing `qualification_templates` table with a `status` field,  
**so that** the workflow builder can display published qualification templates as document requirements and admins can manage document template lifecycles.

---

## Acceptance Criteria

1. The `qualification_templates` table has a `status` VARCHAR(50) column with values: 'draft', 'published', 'archived'
2. Existing `qualification_templates` records default to 'published' status
3. The `workflow_step_template.document_template_id` column has a foreign key constraint referencing `qualification_templates.id` with ON DELETE RESTRICT
4. The `GET /api/document-templates/published` endpoint queries `qualification_templates` instead of returning empty array
5. The endpoint returns only templates where:
   - `tenant_id = current_user.tenant_id`
   - `status = 'published'`
   - `deleted_at IS NULL`
6. Response format: `[{ id: uuid, label: string }]` where label is `qualification_templates.template_name`
7. Results are sorted alphabetically by `template_name`
8. API validation prevents creating workflow steps with document templates that are:
   - Draft or archived status
   - From another tenant
   - Soft-deleted
9. Database migration is reversible (includes rollback SQL)
10. All unit tests pass with 90%+ coverage for new API logic
11. Integration tests verify FK constraint prevents deletion of in-use qualification templates

### API Specifications

[Source: Story 2.2.7.2 Dev Notes]

**Endpoint:** `GET /api/document-templates/published`

**Authentication:** Required (JWT token via `authenticateToken` middleware)

**Authorization:** Tenant-scoped (uses `req.user.tenantId` from JWT)

**Query Pattern (matching form template endpoint):**
```typescript
const templates = await db
  .select({
    id: qualificationTemplates.id,
    label: qualificationTemplates.templateName,
  })
  .from(qualificationTemplates)
  .where(
    and(
      eq(qualificationTemplates.tenantId, tenantId),
      eq(qualificationTemplates.status, "published"),
      isNull(qualificationTemplates.deletedAt)
    )
  )
  .orderBy(asc(qualificationTemplates.templateName));
```

**Response Format:**
```typescript
[
  { id: "uuid-1", label: "ISO Certification Documents" },
  { id: "uuid-2", label: "W-9 Tax Forms" }
]
```

**Error Handling:**
- 401: Unauthorized (no valid JWT token)
- 500: Internal server error (database error)

**Validation Logic (for workflow step creation/update):**

When `document_template_id` is provided in step creation/update:
1. Verify template exists in database
2. Verify template belongs to same tenant (`tenant_id` match)
3. Verify template status is 'published' (not draft or archived)
4. Verify template is not soft-deleted (`deleted_at IS NULL`)

If validation fails, return 400 error with descriptive message.
