## Story

**As a** procurement manager,  
**I want** to initiate a qualification workflow for a supplier,  
**so that** we can formally assess and approve them.

---

## Acceptance Criteria

1. Supplier detail page includes "Start Qualification" button (visible when supplier is in Prospect status)
2. Clicking "Start Qualification" opens workflow initiation modal
3. Modal displays: Supplier name, Selected checklist template dropdown (defaults to tenant default), Risk assessment section, Notes field
4. Risk assessment includes manual inputs: Geographic risk (Low/Medium/High), Financial risk, Quality risk, Delivery risk
5. Overall risk score calculated automatically from individual risk inputs (weighted average)
6. "Initiate Workflow" button creates workflow record in Draft status
7. Workflow creation associates default document checklist with workflow (snapshot, not reference)
8. Workflow displays on supplier detail page in new "Qualifications" tab
9. Workflow status badge shows current stage: Draft, Stage 1 (Pending), Stage 2 (Pending), Stage 3 (Pending), Approved, Rejected
10. Initiator receives confirmation toast: "Qualification workflow initiated for [Supplier Name]"
11. Audit log records workflow initiation with user, timestamp, and initial risk scores
12. Can only initiate one active qualification per supplier at a time (prevents duplicates)

### Previous Story Context

**From Story 2.1 (Qualification Workflow Data Model & Foundation):**
- **Database Schema**: `qualification_workflows` table exists with fields: id, tenant_id, supplier_id, status, initiated_by, initiated_date, current_stage, risk_score
- **Workflow Status Enum**: Draft, Stage1, Stage2, Stage3, Approved, Rejected
- **Risk Score Field**: `numeric(4, 2)` - stores calculated risk score (1.00-3.00 scale)
- **Indexes**: Composite indexes on (tenant_id, supplier_id) and (tenant_id, status)
- **RLS Policies**: Applied to qualification_workflows table for tenant isolation
- **workflow_documents Table**: Links workflows to documents with status tracking (Pending, Uploaded, Approved, Rejected)

**From Story 2.2 (Document Checklist Configuration):**
- **Document Checklists**: Templates defined in `document_checklists` table with JSONB `required_documents` field
- **Checklist Structure**: Array of objects with name, description, required (boolean), type fields
- **Default Template**: Each tenant has one default template (is_default=true)
- **Template Snapshotting**: AC 11 of Story 2.2 states that changes to templates should not affect in-progress qualifications
- **Important**: This story (2.3) must implement the snapshotting logic by copying the checklist's `required_documents` JSONB when creating the workflow

**Route Pattern Example:**
```typescript
import { Elysia, t } from "elysia";
import { authenticate } from "../../lib/rbac/middleware";
import { db } from "../../lib/db";
import { qualificationWorkflows, workflowDocuments } from "@supplex/db";
import { eq, and, isNull, notInArray } from "drizzle-orm";
import { UserRole } from "@supplex/types";

export const initiateWorkflowRoute = new Elysia({ prefix: "/workflows" })
  .use(authenticate)
  .post("/initiate", async ({ body, user, set }: any) => {
    // Check role: Procurement Manager or Admin
    if (!user?.role || (user.role !== UserRole.PROCUREMENT_MANAGER && user.role !== UserRole.ADMIN)) {
      set.status = 403;
      return {
        success: false,
        error: {
          code: "FORBIDDEN",
          message: "Access denied. Required role: Procurement Manager or Admin",
          timestamp: new Date().toISOString(),
        },
      };
    }

    try {
      const tenantId = user.tenantId as string;
      
      // Verify supplier exists and is Prospect status
      const supplier = await db.query.suppliers.findFirst({
        where: and(
          eq(suppliers.id, body.supplierId),
          eq(suppliers.tenantId, tenantId)
        ),
      });
      
      if (!supplier) {
        set.status = 404;
        return { success: false, error: { code: "NOT_FOUND", message: "Supplier not found" } };
      }
      
      if (supplier.status !== "prospect") {
        set.status = 400;
        return { success: false, error: { code: "INVALID_STATUS", message: "Supplier must be in Prospect status" } };
      }
      
      // Check for active workflow (AC 12)
      const activeWorkflow = await db.query.qualificationWorkflows.findFirst({
        where: and(
          eq(qualificationWorkflows.supplierId, body.supplierId),
          eq(qualificationWorkflows.tenantId, tenantId),
          notInArray(qualificationWorkflows.status, ["Approved", "Rejected"])
        ),
      });
      
      if (activeWorkflow) {
        set.status = 409;
        return { success: false, error: { code: "CONFLICT", message: "Supplier already has an active qualification workflow" } };
      }
      
      // Calculate risk score (AC 5)
      const riskValues = { low: 1, medium: 2, high: 3 };
      const riskScore = (
        riskValues[body.riskAssessment.geographic] * 0.3 +
        riskValues[body.riskAssessment.financial] * 0.25 +
        riskValues[body.riskAssessment.quality] * 0.3 +
        riskValues[body.riskAssessment.delivery] * 0.15
      ).toFixed(2);
      
      // Fetch checklist template
      const checklist = await db.query.documentChecklists.findFirst({
        where: and(
          eq(documentChecklists.id, body.checklistId),
          eq(documentChecklists.tenantId, tenantId)
        ),
      });
      
      // Create workflow with snapshotted checklist (AC 7)
      const newWorkflow = await db.insert(qualificationWorkflows).values({
        tenantId,
        supplierId: body.supplierId,
        status: "Draft",
        currentStage: 0,
        riskScore,
        initiatedBy: user.id,
        initiatedDate: new Date(),
        // Note: Snapshot checklist data in metadata or linked table
      }).returning();
      
      // Create workflow_documents for each checklist item
      const workflowDocs = checklist.requiredDocuments.map((item: any) => ({
        workflowId: newWorkflow[0].id,
        checklistItemId: item.id || crypto.randomUUID(), // Generate ID if not present
        status: "Pending",
      }));
      await db.insert(workflowDocuments).values(workflowDocs);
      
      // Update supplier status: Prospect → Qualified
      await db.update(suppliers)
        .set({ status: "qualified", updatedAt: new Date() })
        .where(eq(suppliers.id, body.supplierId));
      
      // Create audit log (AC 11)
      await db.insert(auditLogs).values({
        tenantId,
        userId: user.id,
        action: "workflow_initiated",
        entityType: "qualification_workflow",
        entityId: newWorkflow[0].id,
        metadata: {
          supplierId: body.supplierId,
          riskScore,
          riskAssessment: body.riskAssessment,
        },
      });
      
      set.status = 201;
      return { success: true, data: { workflow: newWorkflow[0] } };
      
    } catch (error: any) {
      console.error("Error initiating workflow:", error);
      set.status = 500;
      return { success: false, error: { code: "INTERNAL_ERROR", message: "Failed to initiate workflow" } };
    }
  }, {
    body: t.Object({
      supplierId: t.String({ format: "uuid" }),
      checklistId: t.String({ format: "uuid" }),
      riskAssessment: t.Object({
        geographic: t.Union([t.Literal("low"), t.Literal("medium"), t.Literal("high")]),
        financial: t.Union([t.Literal("low"), t.Literal("medium"), t.Literal("high")]),
        quality: t.Union([t.Literal("low"), t.Literal("medium"), t.Literal("high")]),
        delivery: t.Union([t.Literal("low"), t.Literal("medium"), t.Literal("high")]),
      }),
      notes: t.Optional(t.String()),
    }),
  });
```

#### UI Component Pattern
[Source: apps/web/app/components/checklists/ChecklistDialog.tsx]

**Modal Dialog Pattern:**
```typescript
import { Dialog, DialogContent, DialogHeader, DialogTitle, DialogDescription } from "~/components/ui/dialog";
import { useRevalidator } from "@remix-run/react";
import { createClientEdenTreatyClient } from "~/lib/api-client";
import { useToast } from "~/hooks/useToast";

export function InitiateWorkflowDialog({ open, onOpenChange, supplier, checklists, token }) {
  const [isSubmitting, setIsSubmitting] = useState(false);
  const revalidator = useRevalidator();
  const { toast } = useToast();

  const handleSubmit = async (data: any) => {
    setIsSubmitting(true);
    try {
      const client = createClientEdenTreatyClient(token);
      const response = await client.api.workflows.initiate.post(data);
      
      if (response?.error) {
        throw new Error(response.error.message || "Failed to initiate workflow");
      }
      
      toast({
        title: `Qualification workflow initiated for ${supplier.name}`,
        variant: "success",
      });
      
      revalidator.revalidate();
      onOpenChange(false);
    } catch (error: any) {
      toast({
        title: "Error",
        description: error.message || "Failed to initiate workflow",
        variant: "destructive",
      });
    } finally {
      setIsSubmitting(false);
    }
  };

  return (
    <Dialog open={open} onOpenChange={onOpenChange}>
      <DialogContent className="max-w-2xl">
        <DialogHeader>
          <DialogTitle>Initiate Qualification Workflow</DialogTitle>
          <DialogDescription>Start a formal qualification process for {supplier.name}</DialogDescription>
        </DialogHeader>
        <InitiateWorkflowForm
          supplier={supplier}
          checklists={checklists}
          isSubmitting={isSubmitting}
          onSubmit={handleSubmit}
          onCancel={() => onOpenChange(false)}
        />
      </DialogContent>
    </Dialog>
  );
}
```

#### Risk Score Calculation (AC 5)
**Formula:**
- Geographic Risk: 30% weight
- Financial Risk: 25% weight
- Quality Risk: 30% weight
- Delivery Risk: 15% weight

**Risk Level Values:**
- Low = 1
- Medium = 2
- High = 3

**Calculation:**
```typescript
riskScore = (geographic * 0.3) + (financial * 0.25) + (quality * 0.3) + (delivery * 0.15)
```

**Range:** 1.00 (all Low) to 3.00 (all High)

**Color Indicators:**
- Green: < 1.5 (Low overall risk)
- Yellow: 1.5 - 2.5 (Medium overall risk)
- Red: > 2.5 (High overall risk)

#### Checklist Snapshotting (AC 7)
**Critical Implementation Note:**

This story MUST implement checklist template snapshotting to satisfy AC 11 from Story 2.2:
> "Changes to templates do not affect in-progress qualifications (snapshot template at workflow creation)"

**Implementation Options:**
1. **Option A**: Store snapshotted checklist in workflow metadata JSONB field
2. **Option B**: Copy `required_documents` JSONB from checklist template to workflow record
3. **Option C**: Create separate `workflow_checklist_snapshot` table

**Recommended**: Option B - Store in workflow's metadata or add `snapshotted_checklist` JSONB field

**Workflow:**
1. When initiating workflow, fetch selected checklist template
2. Copy the `required_documents` JSONB array from template
3. Store copy in workflow record (not a reference to template)
4. Create `workflow_documents` records based on snapshotted data
5. Future edits to template will not affect this workflow's checklist

#### Supplier Status Transitions
[Source: packages/types/src/models/supplier.ts]

**Status Enum:**
- PROSPECT = "prospect" (initial status)
- QUALIFIED = "qualified" (in-progress qualification)
- APPROVED = "approved" (qualification complete)
- CONDITIONAL = "conditional" (approved with conditions)
- BLOCKED = "blocked" (failed qualification or compliance issue)

**Status Transition for This Story:**
- When workflow is initiated: Prospect → Qualified
- Workflow in Draft status: Supplier remains Qualified
- Workflow completion (Story 2.7): Qualified → Approved or Prospect (if rejected)
