## Story

**As a** procurement manager (reviewer),  
**I want to** review and approve or reject stage 1 qualifications,  
**so that** only suitable suppliers proceed to quality review.

---

## Acceptance Criteria

1. "My Tasks" queue in navigation shows count of pending workflow reviews assigned to user
2. Task queue displays list of workflows awaiting approval with: Supplier name, Submitted by, Submitted date, Risk score, Days pending
3. Clicking workflow opens qualification review page with full details
4. Review page displays: Supplier information, Risk assessment, Document checklist with all uploaded documents, Notes from previous stages
5. Documents can be viewed inline (PDFs) or downloaded
6. Reviewer can add review comments in text area
7. "Approve" and "Request Changes" buttons displayed at bottom
8. "Approve" button opens confirmation modal, updates stage status to Approved, advances workflow to Stage 2 (Pending)
9. "Request Changes" button opens modal with required comment field, changes stage status to Rejected, returns workflow to Draft status
10. Email notification sent to initiator: "Qualification for [Supplier] approved/rejected by [Reviewer]"
11. If rejected, notification includes reviewer comments and link to workflow
12. Timeline updated with approval/rejection action, reviewer name, timestamp, comments
13. If approved, Stage 2 record auto-created and assigned to quality manager (based on tenant config)
14. Audit log records approval/rejection decision with comments

#### Database Schema Context

[Source: packages/db/src/schema/qualification-workflows.ts, qualification-stages.ts]

**qualification_workflows Table:**
```typescript
{
  id: uuid (PK),
  tenant_id: uuid (FK → tenants),
  supplier_id: uuid (FK → suppliers),
  status: varchar (Draft, Stage1, Stage2, Stage3, Approved, Rejected), // CRITICAL: Updated by approval/rejection
  initiated_by: uuid (FK → users),
  initiated_date: timestamp,
  current_stage: integer (0-3), // CRITICAL: Updated by approval (0=Draft, 1=Stage1, 2=Stage2, 3=Stage3)
  risk_score: numeric(4, 2),
  snapshotted_checklist: jsonb (RequiredDocumentItem[]),
  created_at: timestamp,
  updated_at: timestamp,
  deleted_at: timestamp
}

Relations:
- supplier: one-to-one with suppliers table (via supplier_id)
- initiator: one-to-one with users table (via initiated_by)
```

**qualification_stages Table:**
```typescript
{
  id: uuid (PK),
  workflow_id: uuid (FK → qualification_workflows),
  stage_number: integer (1-3), // CRITICAL: This story reviews stage_number = 1, creates stage_number = 2
  stage_name: varchar(100), // "Procurement Review", "Quality Review", "Management Approval"
  assigned_to: uuid (FK → users), // CRITICAL: Current user must match to review
  status: varchar (Pending, Approved, Rejected), // CRITICAL: Updated by approval/rejection
  reviewed_by: uuid (FK → users, nullable), // CRITICAL: Set to current user on review
  reviewed_date: timestamp (nullable), // CRITICAL: Set to NOW() on review
  comments: text (nullable), // CRITICAL: Required for rejection, optional for approval
  attachments: jsonb (default []),
  created_at: timestamp,
  updated_at: timestamp,
  deleted_at: timestamp
}
```

**Indexes:**
- `idx_qualification_stages_workflow_stage` on (workflow_id, stage_number) - For retrieving stages sequentially
- `idx_qualification_stages_assigned_to_pending` on (assigned_to) WHERE status='Pending' - For "My Tasks" queue (AC 1, 2)

**suppliers Table:**
```typescript
{
  id: uuid (PK),
  tenant_id: uuid (FK → tenants),
  name: varchar(200),
  tax_id: varchar(50),
  category: varchar(50),
  status: varchar(50), // CRITICAL: Reverted to "Prospect" on rejection
  contact_name: varchar(200),
  contact_email: varchar(255),
  contact_phone: varchar(50),
  address: jsonb,
  risk_score: numeric(4, 2),
  ...
}
```

**Status Transition Logic:**

**On Approval (Stage 1):**
1. qualification_stages[stage_1]: status = "Approved", reviewed_by = current user, reviewed_date = NOW()
2. qualification_workflows: status = "Stage2", current_stage = 2
3. qualification_stages[stage_2]: Create new record with stage_number = 2, assigned_to = quality manager, status = "Pending"
4. Supplier status: Remains "Qualified" (no change)

**On Rejection (Stage 1):**
1. qualification_stages[stage_1]: status = "Rejected", reviewed_by = current user, reviewed_date = NOW(), comments = rejection reason
2. qualification_workflows: status = "Draft", current_stage = 0
3. suppliers: status = "Prospect" (revert from "Qualified")
4. No Stage 2 record created


**ElysiaJS Implementation Pattern:**
```typescript
import { Elysia, t } from "elysia";
import { authenticate } from "~/middleware/authenticate";
import { db } from "@supplex/db";
import { qualificationWorkflows, qualificationStages, suppliers } from "@supplex/db/schema";
import { eq, and, isNull } from "drizzle-orm";

export const approveStageRoute = new Elysia()
  .use(authenticate)
  .post("/:id/stages/:stageId/approve", async ({ params, user, body, set }: any) => {
    // Role check with null-safe pattern
    if (!user?.role || (user.role !== "procurement_manager" && user.role !== "admin")) {
      set.status = 403;
      return { success: false, error: { code: "FORBIDDEN", message: "Access denied" } };
    }
    
    // Verify stage exists and is assigned to current user
    const stage = await db.query.qualificationStages.findFirst({
      where: and(
        eq(qualificationStages.id, params.stageId),
        eq(qualificationStages.workflowId, params.id),
        isNull(qualificationStages.deletedAt)
      ),
      with: {
        workflow: true,
      },
    });
    
    if (!stage) {
      set.status = 404;
      return { success: false, error: { code: "NOT_FOUND", message: "Stage not found" } };
    }
    
    // Check assignment
    if (stage.assignedTo !== user.id) {
      set.status = 403;
      return { success: false, error: { code: "FORBIDDEN", message: "Not assigned to review" } };
    }
    
    // Check status is Pending
    if (stage.status !== "Pending") {
      set.status = 400;
      return { success: false, error: { code: "INVALID_STATE", message: "Stage already reviewed" } };
    }
    
    // Verify tenant isolation
    if (stage.workflow.tenantId !== user.tenantId) {
      set.status = 403;
      return { success: false, error: { code: "FORBIDDEN", message: "Access denied" } };
    }
    
    // Get Stage 2 reviewer
    const stage2Reviewer = await getStage2Reviewer(user.tenantId);
    if (!stage2Reviewer) {
      set.status = 500;
      return { success: false, error: { code: "NO_REVIEWER", message: "No reviewer available for Stage 2" } };
    }
    
    // Begin transaction
    const result = await db.transaction(async (tx) => {
      // Update current stage to Approved
      const approvedStage = await tx.update(qualificationStages)
        .set({
          status: "Approved",
          reviewedBy: user.id,
          reviewedDate: new Date(),
          comments: body.comments || null,
          updatedAt: new Date(),
        })
        .where(eq(qualificationStages.id, stage.id))
        .returning();
      
      // Update workflow to Stage2
      const updatedWorkflow = await tx.update(qualificationWorkflows)
        .set({
          status: "Stage2",
          currentStage: 2,
          updatedAt: new Date(),
        })
        .where(eq(qualificationWorkflows.id, stage.workflowId))
        .returning();
      
      // Create Stage 2 record
      const nextStage = await tx.insert(qualificationStages).values({
        workflowId: stage.workflowId,
        stageNumber: 2,
        stageName: "Quality Review",
        assignedTo: stage2Reviewer.id,
        status: "Pending",
      }).returning();
      
      // Create audit log entry (placeholder for Story 2.10)
      // await createAuditLog(tx, { ... });
      
      return { 
        workflow: updatedWorkflow[0], 
        approvedStage: approvedStage[0],
        nextStage: nextStage[0],
      };
    });
    
    // Queue email notification (stub for Story 2.8)
    await sendStageApprovedEmail(
      stage.workflowId,
      stage.workflow.initiator.email,
      stage.workflow.initiator.fullName,
      stage.workflow.supplier.name,
      user.fullName,
      1,
      "Quality Review"
    );
    
    set.status = 200;
    return { success: true, data: result };
  }, {
    params: t.Object({
      id: t.String({ format: "uuid" }),
      stageId: t.String({ format: "uuid" }),
    }),
    body: t.Object({
      comments: t.Optional(t.String()),
    }),
  });

// Helper function to get Stage 2 reviewer (similar to getStage1Reviewer from Story 2.5)
async function getStage2Reviewer(tenantId: string) {
  // Try tenant settings first
  const tenant = await db.query.tenants.findFirst({
    where: eq(tenants.id, tenantId),
  });
  
  if (tenant?.settings?.workflowReviewers?.stage2) {
    const reviewer = await db.query.users.findFirst({
      where: and(
        eq(users.id, tenant.settings.workflowReviewers.stage2),
        eq(users.tenantId, tenantId),
        eq(users.isActive, true)
      ),
    });
    if (reviewer) return reviewer;
  }
  
  // Fallback: First quality manager
  const qualityManager = await db.query.users.findFirst({
    where: and(
      eq(users.tenantId, tenantId),
      eq(users.role, "quality_manager"),
      eq(users.isActive, true)
    ),
  });
  if (qualityManager) return qualityManager;
  
  // Fallback: First admin
  const admin = await db.query.users.findFirst({
    where: and(
      eq(users.tenantId, tenantId),
      eq(users.role, "admin"),
      eq(users.isActive, true)
    ),
  });
  return admin;
}
```

**CRITICAL ElysiaJS Rules:**
[Source: docs/architecture/coding-standards.md#critical-fullstack-rules]

- **Route Organization**: Child routes (like approve-stage.ts) must NOT have prefix. Parent `index.ts` provides `/workflows` prefix.
- **Parameter Names**: Use consistent parameter names: `:id` for workflow ID, `:stageId` for stage ID
- **Validation**: Use TypeBox (`t.*`) exclusively, never Zod (`z.*`)
- **Authentication**: Use `authenticate` middleware directly on route, not nested in wrappers
- **Role Checks**: Perform inside handler with null checks: `if (!user?.role || ...)`
- **Enum Validation**: Use `t.Union([t.Literal("Approved"), t.Literal("Rejected")])` not Zod schemas

```typescript
export async function loader(args: LoaderFunctionArgs) {
  const { session } = await requireAuth(args);
  const { id } = args.params;
  if (!id) throw new Response("Workflow ID required", { status: 400 });
  
  const token = session?.access_token;
  if (!token) throw new Response("Unauthorized", { status: 401 });
  const client = createEdenTreatyClient(token);
  
  try {
    // Fetch review data (will return 403 if not assigned to current user)
    const reviewResponse = await client.api.workflows[id].review.get();
    
    if (reviewResponse.error) {
      if (reviewResponse.status === 403) {
        // Not assigned to this workflow - redirect to tasks with error
        throw redirect("/tasks?error=not_assigned");
      }
      if (reviewResponse.status === 404) {
        throw new Response("Workflow not found", { status: 404 });
      }
      throw new Response("Failed to load review data", { status: 500 });
    }
    
    return json({
      workflow: reviewResponse.data.workflow,
      supplier: reviewResponse.data.supplier,
      documents: reviewResponse.data.documents,
      stage: reviewResponse.data.stage,
      initiator: reviewResponse.data.initiator,
      token,
    });
  } catch (error) {
    if (error instanceof Response) throw error;
    throw new Response("Failed to load workflow review", { status: 500 });
  }
}
```
