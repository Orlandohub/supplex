## Story

**As an** admin,  
**I want** form and document template selection to show only published templates from my company,  
**so that** I only see relevant templates and cannot accidentally select templates from other tenants.

---

## Acceptance Criteria

1. When adding a workflow step with form integration, form template selection is a dropdown
2. The form template dropdown shows only form templates where:
   - `tenant_id = current_user.tenant_id`
   - `status = 'published'`
   - `deleted_at IS NULL`
3. Dropdown displays `form_template.name` with `form_template_version.version` (e.g., "Supplier Profile v3")
4. Dropdown value is `form_template_version_id` (used to link step to specific form version)
5. When adding a workflow step with document integration, document template selection is a dropdown
6. The document template dropdown shows only document templates where:
   - `tenant_id = current_user.tenant_id`
   - `status = 'published'`
   - `deleted_at IS NULL`
7. If no published templates exist, dropdown shows "No published templates available" message
8. Dropdown options are sorted alphabetically by name
9. API endpoints enforce tenant isolation (cannot fetch templates from other tenants)

## Dev Notes

### Previous Story Context

[Source: Story 2.2.7 - Backoffice Workflow Template Builder]

Story 2.2.7 implements the workflow builder UI where admins configure workflow steps. Each step can optionally include:
- A form (via `form_template_version_id` + `form_action_mode`)
- A document requirement (via `document_template_id` + `document_action_mode`)

User feedback from manual testing revealed that form/document template selection should be tenant-scoped dropdowns showing only published templates, not free-text inputs.

### Current Form Template Schema

[Source: Story 2.2.2 - Form Template Data Model]

```sql
CREATE TABLE form_template (
    id UUID PRIMARY KEY,
    tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    status VARCHAR(50) NOT NULL DEFAULT 'draft', -- 'draft', 'published', 'archived'
    ...
);

CREATE TABLE form_template_version (
    id UUID PRIMARY KEY,
    form_template_id UUID NOT NULL REFERENCES form_template(id) ON DELETE CASCADE,
    tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    version INTEGER NOT NULL,
    status VARCHAR(50) NOT NULL DEFAULT 'draft',
    is_published BOOLEAN NOT NULL DEFAULT false,
    ...
);
```

### Workflow Step Template Schema

[Source: Story 2.2.6 - Workflow Template Data Model]

```sql
CREATE TABLE workflow_step_template (
    id UUID PRIMARY KEY,
    workflow_template_version_id UUID NOT NULL,
    tenant_id UUID NOT NULL,
    step_order INTEGER NOT NULL,
    name VARCHAR(255) NOT NULL,
    
    -- Form integration
    form_template_version_id UUID REFERENCES form_template_version(id) ON DELETE RESTRICT,
    form_action_mode VARCHAR(50), -- 'fill_out' or 'validate'
    
    -- Document integration
    document_template_id UUID, -- Not yet implemented
    document_action_mode VARCHAR(50), -- 'upload' or 'validate'
    ...
);
```

### API Design Pattern

Follow existing pattern from form builder:
- Endpoint: `GET /api/form-templates/published`
- Authentication: Required (get tenant_id from JWT)
- Query: Filter by tenant_id + status='published' + deleted_at IS NULL
- Response: Array of `{ id: string, label: string }`
- Sorted: Alphabetically by name

### UI Component Pattern

Use existing dropdown component from UI library (Shadcn/Radix):

```tsx
<Select
  value={formTemplateVersionId}
  onValueChange={setFormTemplateVersionId}
>
  <SelectTrigger>
    <SelectValue placeholder="Select form template..." />
  </SelectTrigger>
  <SelectContent>
    {formTemplates?.map((template) => (
      <SelectItem key={template.id} value={template.id}>
        {template.label}
      </SelectItem>
    ))}
  </SelectContent>
</Select>
```

### Tenant Isolation Enforcement

Critical: API must enforce tenant isolation at multiple layers:
1. Query filter: `WHERE tenant_id = $1` (from JWT)
2. RLS policy: Supabase Row Level Security as backup
3. Validation: On step creation, verify form_template_version_id belongs to tenant
