### Remix Layout Route Pattern

[Source: docs/architecture/remix-patterns.md, Remix documentation]

**Layout Route Structure:**

To create a persistent app shell that wraps all authenticated pages, use Remix layout routes:

1. Create `apps/web/app/routes/_app.tsx` as a layout route
2. Layout route exports a component that renders `<Outlet />`
3. All child routes use `_app.routename.tsx` naming convention
4. Layout route can have its own loader for shared data

**Example Pattern:**
```typescript
// apps/web/app/routes/_app.tsx
import { Outlet } from "@remix-run/react";
import { AppShell } from "~/components/layout/AppShell";

export default function AppLayout() {
  return (
    <AppShell>
      <Outlet />
    </AppShell>
  );
}
```

**File Naming for Nested Routes:**
- `_app._index.tsx` → `/` (dashboard inside app shell)
- `_app.suppliers._index.tsx` → `/suppliers` (suppliers list inside app shell)
- `_app.suppliers.$id.tsx` → `/suppliers/:id` (supplier detail inside app shell)

**Important:** Routes NOT prefixed with `_app.` will render WITHOUT the app shell (e.g., login, signup pages).

### Sidebar Collapse State Management

[Source: docs/architecture/tech-stack.md]

**Use Zustand for Sidebar State:**

Sidebar collapse state should persist across page loads and be accessible globally.

```typescript
// apps/web/app/stores/navigationStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface NavigationState {
  isSidebarCollapsed: boolean;
  toggleSidebar: () => void;
  setSidebarCollapsed: (collapsed: boolean) => void;
}

export const useNavigationStore = create<NavigationState>()(
  persist(
    (set) => ({
      isSidebarCollapsed: false,
      toggleSidebar: () => set((state) => ({ isSidebarCollapsed: !state.isSidebarCollapsed })),
      setSidebarCollapsed: (collapsed) => set({ isSidebarCollapsed: collapsed }),
    }),
    {
      name: 'navigation-storage', // localStorage key
    }
  )
);
```

**Usage in Components:**
```typescript
const { isSidebarCollapsed, toggleSidebar } = useNavigationStore();
```

**Why Zustand:**
- Lightweight (1KB)
- No boilerplate
- Built-in persistence middleware
- Better than React Context for global UI state

### Accessibility Requirements

[Source: docs/front-end-spec/7-accessibility-requirements.md]

**WCAG 2.1 AA Compliance:**

1. **Color Contrast:** Text must have 4.5:1 contrast ratio with background
   - Tailwind: Use `text-gray-900` on `bg-white` (21:1 ratio ✓)
   - Active states: `text-blue-600` on `bg-blue-50` (7.9:1 ratio ✓)

2. **Keyboard Navigation:**
   - All interactive elements must be reachable via Tab key
   - Visual focus indicators required (Tailwind: `focus:ring-2 focus:ring-blue-500`)
   - Logical tab order: Logo → Search → Notifications → User Menu → Sidebar → Main Content

3. **ARIA Labels:**
   - `<nav role="navigation" aria-label="Main navigation">`
   - `<button aria-label="Toggle sidebar" aria-expanded={!collapsed}>`
   - `<a aria-current="page">` for active menu item
   - `<button aria-label="Search" aria-keyshortcuts="Control+K">`

4. **Screen Reader Support:**
   - Visually hidden text for icon-only buttons
   - Semantic HTML: `<nav>`, `<header>`, `<main>`, `<aside>`
   - Skip link: `<a href="#main-content" class="sr-only focus:not-sr-only">Skip to main content</a>`

5. **Focus Management:**
   - Trap focus within open modals/dropdowns
   - Return focus to trigger button when closing
   - Don't remove focus outline (never use `outline-none` without alternative)

**Tailwind Screen Reader Utilities:**
- `sr-only`: Visually hidden, available to screen readers
- `focus:not-sr-only`: Become visible when focused (for skip links)
