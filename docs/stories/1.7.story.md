## Story

**As a** procurement manager,  
**I want** to create new suppliers and edit existing ones,  
**so that** I can maintain accurate supplier information.

---

## Acceptance Criteria

1. "Add Supplier" button on supplier list navigates to create form
2. Create/Edit form includes fields: Company name (required), Status (dropdown), Address (street, city, state, postal, country), Phone, Email (validated format), Website (validated URL format), Categories (multi-select), Notes (textarea)
3. Primary contact section includes fields: Name (required), Email (validated format, required), Phone (optional)
4. Form validation using React Hook Form + Zod with inline error messages
5. "Save" button disabled until required fields are valid
6. Duplicate detection warns if supplier name matches existing supplier (fuzzy match)
7. Successful save redirects to supplier detail page with success toast notification
8. Cancel button with confirmation if form has unsaved changes
9. Form data persists in browser local storage on blur (auto-save draft) to prevent data loss
10. Edit form pre-populates all fields with existing supplier data
11. Audit trail records who created/edited and when
12. Mobile-responsive form with appropriate input types (email keyboard, phone keyboard)

```typescript
export enum SupplierCategory {

  contactName: string;
  contactEmail: string;
  contactPhone: string;
  address: SupplierAddress;
  certifications: SupplierCertification[];
  metadata: Record<string, any>;
```

**Database Schema:**
[Source: architecture/database-schema.md]
- Table name: `suppliers` (defined in `packages/db/src/schema/suppliers.ts`)
- UUID primary key with `uuid_generate_v4()` default
- Tenant isolation via `tenant_id` foreign key (NOT NULL, CASCADE DELETE)
- Status field stored as VARCHAR(50) enum type
- Category field stored as VARCHAR(50) enum type
- Address stored as JSONB field (must validate all required subfields)
- Certifications stored as JSONB array (can be empty array `[]`)
- Soft delete via `deleted_at` timestamp (must exclude from queries with `WHERE deleted_at IS NULL`)
- Indexes on: `tenant_id`, `name`, `updated_at`, `status`, `category` for performance
- Unique constraint on `(tenant_id, tax_id)` to prevent duplicate tax IDs within same tenant

**Important Note on Contacts:**
- **Current Schema:** Supplier has only ONE primary contact (contactName, contactEmail, contactPhone as flat fields in suppliers table)
- **Implementation:** AC #3 reflects single contact MVP approach. The contact fields (name, email, phone) map directly to the flat contact_* columns in the database schema.
- **Future Enhancement:** Multi-contact support with additional fields (title, isPrimary) via contacts JSONB array is deferred to Story 1.11 to avoid blocking MVP with database migration.
- **Developer Note:** Add TODO comment in ContactFields component referencing future multi-contact enhancement.

### API Endpoint Specifications

[Source: architecture/api-specification.md]

**POST /api/suppliers**
- **Purpose:** Create new supplier
- **Auth:** Requires Admin or Procurement Manager role
- **Tenant Scoping:** Automatically set tenant_id from authenticated user's JWT
- **Request Body:**
```typescript
{
  name: string; // required
  taxId: string; // required
  category: SupplierCategory; // required
  status: SupplierStatus; // required, default "prospect"
  contactName: string; // required
  contactEmail: string; // required, email format
  contactPhone?: string; // optional
  address: SupplierAddress; // required (all subfields)
  website?: string; // optional, URL format
  certifications?: SupplierCertification[]; // optional, default []
  metadata?: Record<string, any>; // optional, default {}
  notes?: string; // optional (stored in metadata.notes)
  riskScore?: number; // optional, 1-10 scale
  forceSave?: boolean; // optional, to override duplicate warning
}
```
- **Response:**
```typescript
{
  supplier: Supplier // Full supplier object with generated id
}
```
- **Errors:**
  - 400 Bad Request if validation fails (return Zod validation errors)
  - 409 Conflict if duplicate supplier name detected (fuzzy match)
  - 403 Forbidden if user lacks permission (not Admin or Procurement Manager)
  - 401 Unauthorized if not authenticated

**PUT /api/suppliers/:id**
- **Purpose:** Update existing supplier
- **Auth:** Requires Admin or Procurement Manager role
- **Tenant Scoping:** Update only if supplier belongs to user's tenant
- **Request Body:** Same as POST, but all fields optional (partial update)
- **Response:**
```typescript
{
  supplier: Supplier // Updated supplier object
}
```
- **Errors:**
  - 404 Not Found if supplier doesn't exist or belongs to different tenant
  - 400 Bad Request if validation fails
  - 403 Forbidden if user lacks permission
  - 401 Unauthorized if not authenticated

**Implementation Notes:**
- Use Drizzle ORM with `.insert()` for POST, `.update().where()` for PUT
- Automatically set `created_by`, `updated_by`, `created_at`, `updated_at`, `tenant_id` from authenticated user context
- For duplicate detection, query suppliers table: `SELECT name FROM suppliers WHERE tenant_id = ? AND deleted_at IS NULL AND name ILIKE ?` (use fuzzy match with % wildcards)
- Return standardized error responses on failures

### Eden Treaty Client Usage

[Source: architecture/coding-standards.md, Story 1.5 implementation]

**CRITICAL:** Never use direct `fetch()` calls. Always use Eden Treaty client for type safety.

**Example Action Implementation for Create:**
```typescript
import { createEdenTreatyClient } from '~/lib/api-client';
import { requireAuth } from '~/lib/auth';

export async function action({ request }: ActionFunctionArgs) {
  const user = await requireAuth(request);
  const formData = await request.formData();
  
  // Parse form data into supplier object
  const supplierData = {
    name: formData.get('name') as string,
    taxId: formData.get('taxId') as string,
    category: formData.get('category') as SupplierCategory,
    // ... other fields
  };
  
  const client = createEdenTreatyClient(request);
  
  const { data, error } = await client.api.suppliers.post(supplierData);
  
  if (error) {
    if (error.status === 409) {
      // Duplicate detected
      return json({ error: 'duplicate', duplicates: error.data?.duplicates }, { status: 409 });
    }
    if (error.status === 400) {
      // Validation errors
      return json({ errors: error.data?.errors }, { status: 400 });
    }
    throw new Response('Failed to create supplier', { status: 500 });
  }
  
  // Success - redirect to detail page
  return redirect(`/suppliers/${data.supplier.id}`);
}
```

```typescript
    }
    if (error.status === 400) {
      return json({ errors: error.data?.errors }, { status: 400 });
    }
    throw new Response('Failed to update supplier', { status: 500 });
  }
  
  return redirect(`/suppliers/${id}`);
}
```

Eden Treaty provides:
- End-to-end type safety (TypeScript inference from ElysiaJS routes)
- Automatic request/response validation
- Error handling with typed error responses

### Local Storage Auto-Save Implementation

[Source: AC #9]

**Auto-Save Strategy:**
- Use localStorage to persist form data as user types
- Key format: `supplier-form-draft-${mode}-${supplierId || 'new'}`
- Save frequency: Debounced by 500ms to avoid excessive writes
- Data to save: Complete form state as JSON string
- On component mount: Check for existing draft and prompt user to restore
- On successful submit: Clear localStorage draft
- On cancel + discard: Clear localStorage draft

**Example Implementation:**
```typescript
import { useEffect } from 'react';
import { useDebounce } from '~/hooks/useDebounce';

const STORAGE_KEY = `supplier-form-draft-${mode}-${supplierId || 'new'}`;

// Watch form values
const formValues = form.watch();
const debouncedValues = useDebounce(formValues, 500);

// Auto-save to localStorage
useEffect(() => {
  if (debouncedValues) {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(debouncedValues));
  }
}, [debouncedValues]);

// On mount, check for draft
useEffect(() => {
  const draft = localStorage.getItem(STORAGE_KEY);
  if (draft) {
    // Show restore prompt modal
    setShowRestorePrompt(true);
    setDraftData(JSON.parse(draft));
  }
}, []);

// Clear draft on successful submit
const handleSubmit = async (data) => {
  // Submit form...
  localStorage.removeItem(STORAGE_KEY);
};
```

### Duplicate Detection Implementation

[Source: AC #6]

**Fuzzy Match Strategy:**
- Use PostgreSQL `ILIKE` for case-insensitive matching
- Query: `SELECT id, name FROM suppliers WHERE tenant_id = ? AND deleted_at IS NULL AND name ILIKE ?`
- Match pattern: `%${inputName}%` (contains match)
- For better fuzzy matching, consider Levenshtein distance algorithm (can add in Phase 2)
- Only show duplicates within same tenant (never cross-tenant)
- Return up to 5 potential duplicates to avoid overwhelming user
