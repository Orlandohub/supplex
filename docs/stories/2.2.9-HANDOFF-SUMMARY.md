## What Changed?

### The Problem
Original Story 2.2.9 was too narrow - it treated "qualification" as a special, isolated concept. But suppliers need many types of workflows: qualification, audits, onboarding, compliance checks, risk assessments, etc.

### The Solution
**Generalized the workflow system** to support ANY supplier-related process while adding custom status tracking capabilities.

---

## Key Changes at a Glance

### 1. Database (2 new fields)
```sql
-- workflow_template table
ALTER TABLE workflow_template 
ADD COLUMN active BOOLEAN NOT NULL DEFAULT true;

-- workflow_step_template table
ALTER TABLE workflow_step_template 
ADD COLUMN completion_status VARCHAR(100);
```

### 2. Features

#### ✅ Active/Inactive Workflow Templates
- Admins can toggle templates on/off
- Only active templates appear in dropdowns
- Inactive templates can't be instantiated (existing workflows continue)

#### ✅ Custom Workflow Status Tracking
- Admins set `completion_status` on each step (e.g., "Documents Submitted", "Under Review", "Approved")
- When step completes → workflow status updates automatically
- Gives users meaningful status labels (not technical states)

#### ✅ UI Generalization
- "Qualifications" tab → **"Workflows"** tab
- `/qualifications` route → **`/workflows`** route
- "Start Qualification" → **"Start Process"**
- Works for ANY supplier process type

## Example: Custom Status Tracking

**Workflow:** Supplier Qualification

```
Step 1: "Submit Documents"
  completion_status = "Documents Submitted"
  
Step 2: "Initial Review"
  completion_status = "Under Review"
  
Step 3: "Quality Audit"
  completion_status = "Audit in Progress"
  
Step 4: "Final Approval"
  completion_status = "Qualified"
```

**Runtime Behavior:**
- User completes Step 1 → Workflow status: **"Documents Submitted"**
- User completes Step 2 → Workflow status: **"Under Review"**
- User completes Step 3 → Workflow status: **"Audit in Progress"**
- User completes Step 4 → Workflow status: **"Qualified"**

### 3. Key Implementation Detail
**Completion Status Logic** (apps/api/src/lib/workflow-engine/complete-step.ts):

```typescript
// After marking step as completed (line ~120)

// Get step template
const stepTemplate = await db.query.workflowStepTemplate.findFirst({
  where: and(
    eq(workflowStepTemplate.id, stepInstance.workflowStepTemplateId),
    eq(workflowStepTemplate.tenantId, tenantId)
  ),
});

// Copy completion_status to process status (if set)
if (stepTemplate?.completionStatus) {
  await db
    .update(processInstance)
    .set({
      status: stepTemplate.completionStatus,
      updatedAt: new Date(),
    })
    .where(eq(processInstance.id, stepInstance.processInstanceId));
}
```
