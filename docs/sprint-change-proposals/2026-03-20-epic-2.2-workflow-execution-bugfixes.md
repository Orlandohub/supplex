## Issue Summary

During manual acceptance testing of the workflow engine (after Stories 2.2.14 and 2.2.15), 8 issues were identified:

1. **CRITICAL BUG:** Workflow engine breaks after step 1 — next steps get activated but no tasks are created, validation tasks are not created, and the "Complete Step" button does nothing.
2. **HIGH BUG:** After submitting a form in a workflow, the user is not redirected back and the step is not auto-completed.
3. **MEDIUM BUG:** Breadcrumbs on workflow form pages show "Forms > Form" instead of workflow context.
4. **MEDIUM BUG:** Workflow progress visualization doesn't show validation as a visual step.
5. **MEDIUM BUG:** Workflow detail page doesn't show the workflow template name.
6. **LOW BUG:** No loading state when form instances are created on first navigation.
7. **LOW VALIDATION:** Dropdown fields without values should be rejected (already implemented — needs verification).
8. **MEDIUM FEATURE:** Need "Forms" tab on supplier detail page.

### Root Cause

Story 2.2.14 renamed database columns from `workflowTemplateVersionId` to `workflowTemplateId` and metadata keys accordingly. However, several workflow engine files were NOT updated:

- `transition-to-next-step.ts` still reads `workflowTemplateVersionId` from process metadata and queries a non-existent column
- `complete-step.ts` and `approve-validation-task.ts` reference `step.workflowStepTemplateId` which doesn't exist on `step_instance`
- `routes/workflows/steps/complete.ts` and `return-to-previous-step.ts` query step templates without `workflowTemplateId` filter
