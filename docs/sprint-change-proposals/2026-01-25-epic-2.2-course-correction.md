## 1. Executive Summary

### Change Trigger
Manual testing and key user feedback during Story 2.2.7 (Backoffice Workflow Template Builder) revealed two critical issues:

1. **Issue #1 - Form/Document Template Selection Not Tenant-Scoped:**  
   When adding workflow steps, form template and document template inputs are not dropdowns filtered by the user's `tenant_id` and `status = 'published'`. This creates potential security and UX issues.

2. **Issue #2 - Process Type Constraint Should Be Removed:**  
   The current design uses `process_type` field to categorize workflows into system-defined types (e.g., `supplier_qualification`, `sourcing`). This unnecessarily constrains organizations. Users want to create workflows freely based on name and description without software-imposed type restrictions.
